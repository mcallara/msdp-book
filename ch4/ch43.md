# Assignment 2 - AI-Assisted Software Development

In this assignment you will apply the practices from Chapter 4 to build a small
Python command-line tool named `autograde`. The goal is not only to produce a
working program, but to show that you can collaborate with an AI coding agent in
a controlled way.

Your repository should demonstrate four things:

- you can describe behavior before implementation,
- you can turn that behavior into executable tests,
- you can store stable guidance in repository artifacts,
- and you can use AI assistance without giving up review and control.

## Task

Create a public GitHub repository containing a Python project called
`autograde`. The tool should inspect a target repository and report whether a
set of required conditions hold.

At minimum, implement checks for the following conditions:

- the project is public,
- the project is a Git repository,
- the `main` branch exists,
- the `feature` branch exists on the remote,
- `file1.txt` exists on `main`.

You may implement additional checks if you want, but the minimum set above must
work correctly.

## Required process artifacts

Your submission must include evidence that you used the Chapter 4 workflow,
not just the final code.

Include all of the following in the repository:

- a `specs/` directory with at least one numbered spec per implemented check,
- a `tests/` directory with automated tests,
- an `AGENTS.md` file with repository-specific instructions for the agent,
- and at least one saved prompt or equivalent reusable instruction artifact.

The important requirement is that the artifacts are meaningful. They should help
another person, or another future agent session, understand how the project is
supposed to evolve.

## Suggested workflow

For each check, follow this order:

1. Write a short spec describing the behavior.
2. Ask the agent to produce one failing test for that spec.
3. Run the test and confirm it fails for the right reason.
4. Ask for the smallest implementation that makes the test pass.
5. Re-run the tests and refine the repository instructions if you learned a new stable convention.

If you skip directly to implementation, you are missing the main point of the
assignment.

## What to submit

Your final repository should include:

- the Python package or script implementing `autograde`,
- passing tests,
- the required specs and agent artifacts,
- and a short `README.md` explaining how to run the tool and the test suite.

Be prepared to explain which parts of the work were delegated to the agent,
which parts you reviewed manually, and how your specs and tests helped you catch
or prevent mistakes.
