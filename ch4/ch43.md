## 4.3 Managing Python Versions

When you start a Python project, one of the first questions is: which Python version should this project use?

With uv, the goal is not to manage Python versions manually in a separate tool and then hope your project picks the right interpreter. Instead, uv lets you declare which version you want, create the environment with that version, and use it consistently across project commands.

For a beginner, the most useful mental model is this:

- `pyproject.toml` says which Python versions the project supports;
- `.python-version` says which Python version you want to use locally in this project;
- `uv venv` creates the virtual environment with that version.

### The Default Workflow

Suppose you want to work on a project with Python 3.12. A simple workflow is:

```bash
uv python pin 3.12
uv venv
uv run python --version
```

The first command creates a `.python-version` file in the project directory. The second creates a virtual environment using that Python version. The third checks which interpreter the project is actually using.

This is a good default workflow because it is explicit and easy to repeat. Once the version is pinned, later commands such as `uv run` and `uv sync` will reuse it automatically.

```{admonition} What to notice
:class: hint
- `uv python pin 3.12` records your local choice for this project.
- `uv venv` uses that choice to create the virtual environment.
- `uv run python --version` is a quick way to verify that everything is using the interpreter you expect.
```

### Project Requirements vs Local Choice

It is important to distinguish between the Python versions your project supports and the one you are currently using during development.

In `pyproject.toml`, the `requires-python` field declares compatibility. For example, this project may support Python 3.10 or newer:

```toml
[project]
requires-python = ">=3.10"
```

That does not force every developer to use exactly the same patch release. It only says which versions are acceptable for the project.

The `.python-version` file serves a different purpose: it records the specific version you want uv to prefer in your working directory. For example:

```text
3.12
```

This separation is useful. The project can support a range of Python versions, while each developer can still pin one concrete version locally.

### How uv Chooses a Python Interpreter

uv can work with two kinds of Python installations:

- **managed Python installations**, which are downloaded and installed by uv;
- **system Python installations**, which include Python versions already available on your machine, whether they came from the operating system, Homebrew, pyenv, Conda, or some other tool.

By default, uv will look for a compatible Python installation first. If it cannot find one, it can download a managed version automatically.

That means you often do not need to install Python manually before creating the environment. If the pinned or requested version is missing, uv can fetch it for you.

### Requesting a Version Explicitly

Most uv commands accept a `--python` flag. Use this when you want to request a version directly in the command instead of relying on `.python-version`.

For example:

```bash
uv venv --python 3.11.6
```

If Python 3.11.6 is not already available, uv can download it and then create the virtual environment.

You can also use broader requests such as:

```bash
uv venv --python 3.12
uv venv --python '>=3.10,<3.13'
```

In practice, the most useful request formats are:

- an exact version such as `3.12.3`;
- a minor version such as `3.12`;
- a version range such as `>=3.10,<3.13`;
- a direct interpreter path such as `/opt/homebrew/bin/python3.12`.

For day-to-day project work, pinning the version once is usually easier than passing `--python` repeatedly. The flag is most useful when you want to override the default behavior for a single command.

### When to Use `uv python install`

You can ask uv to install Python explicitly with:

```bash
uv python install 3.12
```

This installs the latest supported patch release in the Python 3.12 series.

If you need an exact patch version, request it directly:

```bash
uv python install 3.12.3
```

You can also install several versions at once:

```bash
uv python install 3.10 3.11 3.12
```

Most beginners do not need this command as the first step, because uv can install Python automatically when a command such as `uv venv` requests it. `uv python install` is most useful when you want to prepare versions ahead of time, manage several interpreters explicitly, or work without relying on automatic downloads.

### Inspecting What Is Available

If you want to see which Python versions uv can discover or install, use:

```bash
uv python list
```

To narrow the results to a particular series:

```bash
uv python list 3.12
```

If you only want to see interpreters that are already installed on your machine:

```bash
uv python list --only-installed
```

This is useful when you are unsure which interpreters are already available before pinning or creating an environment.

### Finding the Exact Interpreter Path

If you need the exact path to the interpreter that uv would use, run:

```bash
uv python find
```

This prints the path to the first compatible Python executable that uv discovers.

You can also ask for a specific version:

```bash
uv python find 3.12
uv python find '>=3.11'
```

If you want to ignore virtual environments and search only for system interpreters, use:

```bash
uv python find --system
```

This command is especially helpful when configuring an editor or checking which interpreter uv is about to use.

### Upgrading Managed Python Versions

If you are using a uv-managed Python installation, you can upgrade it to the latest supported patch release with:

```bash
uv python upgrade 3.12
```

Or upgrade all uv-managed Python versions at once:

```bash
uv python upgrade
```

This updates patch releases within the same minor version, for example from Python 3.12.7 to Python 3.12.8.

### Installing Python Executables on Your PATH

uv can also install Python executables such as `python3.12` onto your `PATH`. If the directory used by uv for these executables is not yet available in your shell, you can update your shell configuration with:

```bash
uv python update-shell
```

This is a convenience feature for working with Python outside a specific project. It is not required for the normal project workflow shown earlier in this section.
